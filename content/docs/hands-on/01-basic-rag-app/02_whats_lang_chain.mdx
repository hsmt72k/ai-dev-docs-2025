LangChain は、AI アプリケーションの開発を支援する強力なフレームワークです。
このセクションでは、LangChain の主要なコンセプト、特徴、及びその役割について詳しくみていきます。

---

## LangChain の概要

LangChain の公式ドキュメントでは、フレームワークを以下のようにな表現でシンプルに説明しています。

公式ドキュメント | LangChain<br />
https://js.langchain.com/docs/introduction

```
Chains, agents, and retrieval strategies that make up an application's cognitive architecture
アプリケーションの認知アーキテクチャを構成するチェーン、エージェント、および検索戦略
```

![](/hands-on/01-basic-rag-app/02_whats_lang_chain/lang_chain_docs.png)

LangChain は、AI アプリケーションの「認知アーキテクチャ」を構築するためのフレームワークです。
認知アーキテクチャとは、AI が情報を処理し、応答を生成するための構造や仕組みのことを指します。

LangChain では、以下の主要な要素を提供しており、それらを組み合わせてアプリケーションを設計します。

---

## 1. Chains（チェーン）

- **意味:** チェーンとは、複数の処理を一連の流れでつなげたものです。
  例えば、「ユーザーからの質問 → 関連する情報を検索 → 検索結果を基に応答を生成」という流れがチェーンです

- **具体例:** ユーザーの質問の文脈を解析し、異なるデータベースを検索、その結果を基にAIが適切な回答を生成します

---

## 2. Agents（エージェント）

- **意味:** エージェントは、タスクを実行する主体です。
  指示に従い、状況に応じて適切な処理を行い、LLMが自律的にツールを選択して実行する能力を持ちます

- **具体例:** 「今日の天気」と聞かれた場合、エージェントがWeb検索ツールを使って最新の情報を取得し、それに基づいて回答を生成します

---

## 3. Retrieval Strategies（検索戦略）

- **意味:** 検索戦略は、ユーザーの質問に関連する情報を取得する方法です。
  ベクトルデータベースや API を使用して、情報を効率的に検索し、LLMが正確な応答を生成できるようにします

- **具体例:** 企業情報を検索し、最新の決算情報を取得して応答を生成します

---

## Cognitive Architecture（認知アーキテクチャ）

- **意味:** 認知アーキテクチャは、AI システムがどのように情報を処理し、学び、意思決定を行うかを定義する設計の枠組みです。
  LangChain では、アプリケーションが情報をどのように取得し、処理し、最終的に応答を生成するかを示す「知的な処理の構造」を提供します

## 用語のまとめ

LangChainの主要な用語を簡潔にまとめると以下のようになります：

- **チェーン:** ユーザーからの質問を受け、関連情報を検索し、その結果をもとに回答を生成するプロセス
- **エージェント:** 質問に応じて最適な検索や生成のプロセスを選び、実行するシステム
- **検索戦略:** 質問に関連する情報をどのように取得し、どのデータソースを使うかを決定する方法

## LangChain の役割と目的

LangChain は、これらの要素を組み合わせて、AIアプリケーションの「思考の仕組み」を設計するためのツールセットを提供します。
これにより、開発者は複雑なAIシステムを簡単に構築でき、チャットボット、文書要約、データ分析など、様々な LLM アプリケーションの開発が可能になります。
また、活発なコミュニティも LangChain の強みです。
