LangChain の公式ドキュメントを読むと、LangChain とは何かを次に短い一文で説明している。

```
Chains, agents, and retrieval strategies that make up an application's cognitive architecture
アプリケーションの認知アーキテクチャを構成するチェーン、エージェント、および検索戦略
```

LangChain は、AIアプリケーションの「認知アーキテクチャ」を構築するためのフレームワークである。

この「認知アーキテクチャ」とは、AIが情報を処理し、応答を生成するための構造や仕組みのことを指す。

LangChain は以下の3つの主要な要素を提供し、それらを組み合わせてアプリケーションを設計する。

### 1. Chains（チェーン）

- **意味:** チェーンとは、複数の処理を一連の流れでつなげたもの。
例えば、「ユーザーからの質問 → 関連する情報を検索 → 検索結果を基に応答を生成」という流れが「チェーン」である。
LangChain では、これら処理の流れ（チェーン）を定義して、モデルが必要なタスクを順番にこなせるようにする

- **具体例:** 質問の文脈をまず解析し、ユーザの意図に合わせて異なるデータベースを検索し、得られた結果を基に AI が適切な回答を作り出す

### 2. Agents（エージェント）

- **意味:** エージェントは、タスクを実行する主体のこと。
与えられた指示に従い、状況に応じて適切な処理を行う役割を担う。
エージェントは、LLM が自律的にツールを選択し、実行する能力を持つ。ユーザの入力や環境に応じて、適切なツールを使い分け、タスクを達成する

- **具体例:** 例えば、「今日の天気」と聞かれた場合、エージェントが Web 検索ツールを使って最新情報を取得し、それに基づいて回答する。
また、法律に関する質問に対しては、関連する法令や判例を検索して適切な回答を提供する

### 3. Retrieval Strategies（検索戦略）

- **意味:** 検索戦略とは、ユーザの質問に関連する情報を取得するための方法である。
ベクトルデータベースや API を使用して、情報を効率的に検索し、LLM が正確で適切な応答を生成できるようにする。
LangChainは、様々な検索方法を提供し、アプリケーションの要件に合わせて検索戦略を選択できる

- **具体例:** 企業情報を検索し、そこから最新の決算情報を取得して応答を生成するといった例が挙げられる

これらの要素を組み合わせることで、LangChain は「認知アーキテクチャ」を設計する。

### Cognitive Architecture（認知アーキテクチャ）

- **意味:** 「認知アーキテクチャ」という言葉は、AIシステムがどのように情報を処理し、学び、意思決定を行うかを定義する設計の枠組みを指す。
LangChain の場合、アプリケーションがどのように情報を取得し、処理し、最終的に応答を生成するかという「知的な処理の構造」を示している。

LangChain は、認知アーキテクチャを構築するための抽象化レイヤーを提供し、開発者は複雑な AI システムをより簡単に構築することができる。

### 用語のまとめ

各用語を簡潔にまとめると以下のようになる。

- **チェーン:** ユーザからの質問を受けて、関連情報を検索し、その結果をもとに回答を生成する一連のプロセス

- **エージェント:** 質問に応じて、最適な検索や生成のプロセスを選び、実行するシステム

- **検索戦略:** 質問に関連する情報をどのように取得し、どのデータソースを使うかを決定する方法

### LangChain の役割と目的

LangChain は、これらの要素を組み合わせて、AIアプリケーションの「思考の仕組み」を設計するためのツールセットを提供している。

アプリケーションがどのように情報を取得し、処理し、応答を生成するかを最適化するために、これらの構造を柔軟に設計できるフレームワークである。

LangChain を使うことで、チャットボット、文書要約、データ分析など、様々な LLM アプリケーションを開発することができる。

活発なコミュニティがあることも、LangChain の強みでもある。

では、続いてはいよいよ、アプリの開発環境の構築を進めていく。
