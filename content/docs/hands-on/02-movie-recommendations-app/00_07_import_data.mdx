## データのインポート

データのインポートを進めていきます。

import { Step, Steps } from 'fumadocs-ui/components/steps';

<Steps>
<Step>

#### 1. コレクションの作成

Astra DB にログイン後、「Load Data」をクリックします。

提供されるサンプルコレクションを使用するか、自分で空のコレクションを作成するかを選択できます。
今回は「movies」という名前のコレクションを作成します。

このコレクションはベクトル対応であり、OpenAI との統合が確認できます。

</Step>

<Step>

#### 2. 埋め込みモデルの選択

OpenAI Embeddings モデルを選択します。

古いモデル（例: text-embedding-ada-002）から新しいモデル（例: text-embedding-3-large）まで利用可能です。

次元数（例: 512）を設定します。次元数が高いほど精度が向上しますが、計算コストが増加します。

類似度計算には「コサイン類似度」を選択することが一般的です。

</Step>

<Step>

#### 3. データのインポート

以下の CSV ファイルを Astra DB にドラッグ＆ドロップでインポートします。

import { File, Folder, Files } from "fumadocs-ui/components/files";

<Files>
  <Folder name="src" defaultOpen>
    <Folder name="data" defaultOpen>
      <a href="/hands-on/02-movie-recommendations-app/src/data/import_data.csv" target="_blank">
        <File name="import_data.csv" />
      </a>
    </Folder>
  </Folder>
</Files>

映画情報の「プロット（あらすじ）」フィールドをベクトル化対象として選択します。

Astra DB が自動的にプロット情報をベクトル化し、データセットを作成します。

</Step>

<Step>

#### 4. パッケージのインストール

接続詳細ページで「TypeScriptクライアント」を選択し、必要な依存関係（npm パッケージ）をインストールします。

</Step>

<Step>

#### 4. 接続情報の取得

接続用トークンを生成し、安全に保存してプロジェクトに組み込みます。

</Step>

<Step>

#### 5. ベクトル検索の実行

映画タイトル（例: 「ターミネーター2」）で検索すると、類似映画がスコア付きで表示されます（例: 類似度100%は同じ映画、84%は関連映画）。

</Step>
</Steps>

ベクトル化されたデータは映画プロット間の類似性比較などに利用可能です。

簡単なセットアップで強力な検索機能を実現できます。

### まとめ

このように、Astra DB は簡単な操作で高度なベクトル検索機能を提供し、映画情報の比較や検索に役立ちます。
