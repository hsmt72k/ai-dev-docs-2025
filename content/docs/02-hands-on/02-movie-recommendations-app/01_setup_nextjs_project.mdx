VSCode で開いたら、`Shift + @` キーでターミナルを開く。

Windows Terminal では以下のコマンドを実行して、Next.js プロジェクトを作成する。

```bash title="Terminal"
bunx create-next-app@latest .
```

Bash コマンドでは、ドット「.」はカレントディレクトリを示す。
カレントディレクトリとは、現在自分がいるフォルダのことである。

つまり、「cd ./chat-ai-rag-2025-04」とは、カレントディレクトリにある、
「chat-ai-rag-2025-04」にカレントディレクトリを移動するコマンドである。

「bunx create-next-app@latest .」は、カレントディレクトリに Next.js プロジェクトを作成するコマンドである。

「create-next-app」コマンドにドット「.」オプションを付けないと、カレントディレクトリにプロジェクトディレクトリが作成されてしまう。
これらの挙動の違いは実際にコマンドを実行してみて、結果の差を確認することで Shell への理解を深めてほしい。

「create-next-app」コマンドを実行すると、オプションの選択を求められる。
オプションは以下の通りに選択する。

```shell title="Terminal"
√ Would you like to use TypeScript? ... Yes
√ Would you like to use ESLint? ... Yes
√ Would you like to use Tailwind CSS? ... Yes
√ Would you like your code inside a `src/` directory? ... Yes
√ Would you like to use App Router? (recommended) ... Yes
√ Would you like to use Turbopack for `next dev`? ... Yes
√ Would you like to customize the import alias (`@/*` by default)? ... No
```
